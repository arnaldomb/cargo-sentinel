# Quick Task 260712-t48: Corrigir THUMBNAIL_PROXY_URL em dev local

## Accomplishments

- Diagnosticado: `apps/api/src/services/garage.ts` (`getThumbnailProxyUrl`) monta a URL da miniatura usando `THUMBNAIL_PROXY_URL`, que em `docker-compose.yml` é `"${WEB_PUBLIC_URL}/api/image-proxy"` — `WEB_PUBLIC_URL` no `.env` é o domínio de produção (`https://portal.ggtronic.com.br`). Em dev local (`localhost:3000`) essa URL gerada pelo backend não corresponde ao host que o navegador usa, então as imagens não carregavam (ícone quebrado no feed).
- Confirmado via curl que o endpoint `/api/image-proxy?key=...` em si funciona perfeitamente (retornou JPEG válido de 78KB da câmera real) — o bug era só a URL gerada, não o proxy.
- Corrigido em `docker-compose.override.yml` (serviço `api`): `THUMBNAIL_PROXY_URL: "http://localhost:3000/api/image-proxy"`.
