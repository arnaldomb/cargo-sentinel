## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Intelbras camera (or attacker) ‚Üí /api/lpr webhook | Untrusted multipart/JSON input including a base64 image crosses here |
| API worker ‚Üí Garage / Postgres | Decoded image and event record written to storage |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-01 | Spoofing | Fake webhook POST | mitigate | Worker rejects events whose `CameraId` has no matching `Camera.codigoLpr` row (V4); cryptographic webhook auth deferred to Phase 2 |
| T-1-02 | Denial of Service | Base64 payload bombing | mitigate | `express.json({ limit: '2mb' })` caps body size (Plan 01); normalize layer validates plate/image present before the worker decodes |
| T-1-D2 | Information Disclosure (cross-tenant) | empresaId from payload | mitigate | empresaId resolved ONLY from `camera.empresaId` (trusted DB row), never read from the webhook body |
| T-1-04 | Information Disclosure | Image accessible without auth | mitigate | Photos written to a non-public Garage bucket; access only via 300s presigned GET URLs (STORAGE-03) signed with the public domain |
| T-1-V6 | Tampering | Weak idempotency hash | mitigate | SHA256 (Node `crypto`), never MD5 (V6); two-layer dedup (BullMQ jobId + Prisma upsert) |
</threat_model>
