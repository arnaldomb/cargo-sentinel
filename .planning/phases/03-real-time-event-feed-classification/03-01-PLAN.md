# 03-01 — Modelo `Placa`, auditoria e endpoint de classificação

## Objective

Criar a camada de domínio que falta para a Fase 3: uma entidade `Placa` tenant-scoped com classificação atual, um histórico auditável de mudanças de classificação e uma mutation HTTP para atualizar essa classificação sem depender ainda do dashboard em tempo real.

## Scope

- adicionar `Placa` ao schema Prisma
- adicionar `ClassificacaoHistorico` ao schema Prisma
- relacionar `Evento` com `Placa`
- ajustar o worker LPR para criar/upsertar `Placa`
- expor endpoint autenticado para reclassificação
- persistir log de auditoria com usuário e timestamp

## Out Of Scope

- Socket.IO
- feed visual no frontend
- alerta cross-site
- WhatsApp

## Context

O código atual já grava `Evento` com `placaNumero` e `classificacao`, mas não existe uma fonte de verdade tenant-wide para a placa. Sem isso, a classificação atual fica dispersa nos eventos e não é possível propagar alterações de forma confiável entre dashboard, histórico e regras futuras de inteligência.

## Interfaces

### Prisma

Adicionar:

```prisma
model Placa {
  id                    String         @id @default(cuid())
  numero                String
  empresaId             String
  empresaTransportadora String?
  motorista             String?
  tipoVeiculo           String?
  material              String?
  observacao            String?
  classificacao         Classificacao  @default(VISITANTE)
  createdAt             DateTime       @default(now())
  updatedAt             DateTime       @updatedAt

  empresa               Empresa        @relation(fields: [empresaId], references: [id])
  eventos               Evento[]
  historico             ClassificacaoHistorico[]

  @@unique([numero, empresaId])
  @@index([empresaId, classificacao])
}

model ClassificacaoHistorico {
  id                 String         @id @default(cuid())
  placaId            String
  empresaId          String
  classificacaoDe    Classificacao?
  classificacaoPara  Classificacao
  observacao         String?
  usuarioId          String
  createdAt          DateTime       @default(now())

  placa              Placa          @relation(fields: [placaId], references: [id])
  empresa            Empresa        @relation(fields: [empresaId], references: [id])
  usuario            User           @relation(fields: [usuarioId], references: [id])
}
```

Atualizar `Evento` para incluir:

```prisma
placaId String?
placa   Placa? @relation(fields: [placaId], references: [id])
```

### HTTP

```http
PATCH /api/placas/:placaId/classificacao
```

Payload:

```json
{
  "classificacao": "ATENCAO",
  "observacao": "Texto opcional"
}
```

## Tasks

### Task 1: Expandir schema e gerar client

- editar `packages/database/prisma/schema.prisma`
- incluir `Placa`, `ClassificacaoHistorico` e a relação opcional em `Evento`
- rodar `prisma generate` e `prisma db push`

### Task 2: Ajustar pipeline LPR para upsert de `Placa`

- editar `apps/api/src/jobs/worker.ts`
- antes de gravar `Evento`, fazer `upsert` em `Placa` por `numero + empresaId`
- se a placa ainda não existir, criar com `classificacao = VISITANTE`
- salvar `placaId` e o snapshot da classificação atual em `Evento`

### Task 3: Criar rota de classificação

- criar `apps/api/src/routes/placas.ts`
- montar a rota sob `protectedPipeline`
- permitir `ADMIN_EMPRESA` e `OPERADOR`
- validar enum de entrada
- buscar `Placa` pelo `tenantClient`
- atualizar classificação e observação
- gravar `ClassificacaoHistorico`
- devolver DTO enxuto com classificação nova e metadados de auditoria

### Task 4: Montar testes do domínio

- teste do worker cobrindo criação automática de placa nova com `VISITANTE`
- teste da rota cobrindo mudança de classificação dentro do tenant
- teste da rota cobrindo 404 em placa fora do tenant
- teste da auditoria cobrindo `usuarioId`, `classificacaoDe`, `classificacaoPara`

## Verification

- `pnpm --filter @cargo-sentinel/database exec prisma validate`
- `pnpm --filter @cargo-sentinel/api test -- --run src/routes/placas`
- `pnpm --filter @cargo-sentinel/api test -- --run src/jobs`

## Acceptance Criteria

- nova placa detectada é criada tenant-scoped com classificação `VISITANTE`
- `Placa` é única por `numero + empresaId`
- reclassificação atualiza `Placa.classificacao`
- cada reclassificação cria 1 linha em `ClassificacaoHistorico`
- rota não cruza tenant

## Done

O backend passa a ter uma fonte de verdade para a classificação atual da placa e um histórico auditável que pode ser usado pelo feed em tempo real e pelas fases seguintes.
