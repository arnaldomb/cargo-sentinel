2. Novo model `Relatorio`:
```prisma
model Relatorio {
  id          String           @id @default(cuid())
  empresaId   String
  status      RelatorioStatus  @default(PENDENTE)
  formato     String           // "PDF" | "XLSX"
  filtros     Json             // { dataInicio?, dataFim?, obraId?, cameraId?, classificacao?, placa? }
  garageKey   String?          // preenchido quando PRONTO: reports/{empresaId}/{id}.pdf ou .xlsx
  expiresAt   DateTime?        // now() + 1h ao marcar PRONTO (REPORTS-07)
  erroMsg     String?          // preenchido quando ERRO
  criadoPor   String           // userId do solicitante
  criadoEm   DateTime         @default(now())
  atualizadoEm DateTime       @updatedAt
  empresa     Empresa          @relation(fields: [empresaId], references: [id])
  usuario     User             @relation("RelatoriosUsuario", fields: [criadoPor], references: [id])

  @@index([empresaId, criadoEm(sort: Desc)])  // listagem tenant-scoped (REPORTS-01)
  @@index([criadoPor, criadoEm(sort: Desc)])  // filtrar por usuário
}
```

3. Adicionar relação inversa em `Empresa`:
```prisma
relatorios Relatorio[]
```

4. Adicionar relação inversa em `User`:
```prisma
relatorios Relatorio[] @relation("RelatoriosUsuario")
```

Atenção: User tem `empresaId String?` — Super Admin não tem empresaId. O campo
`criadoPor` em Relatorio é NOT NULL pois relatórios sempre têm dono. `empresaId`
em Relatorio vem do JWT do solicitante, nunca do parâmetro do cliente.

```typescript
 */
async function processReportJob(job: Job<ReportJobPayload>): Promise<void> {
  const { relatorioId } = job.data;
  console.log(`[report-worker] processando relatorio ${relatorioId} — stub`);
  // Implementação completa no plan 06-02
  throw new Error(`report-worker stub: relatorio ${relatorioId} não implementado ainda`);
}

let workerInstance: Worker | null = null;

/**
 * Starts the report worker with concurrency 2.
 * Call once from app bootstrap (e.g., src/index.ts or src/app.ts).
 */
export function startReportWorker(): Worker {
  if (workerInstance) return workerInstance;

  workerInstance = new Worker<ReportJobPayload>('report-jobs', processReportJob, {
    connection: createRedisConnection(),
    concurrency: 2,
  });

  workerInstance.on('completed', (job) => {
    console.log(`[report-worker] job ${job.id} concluído`);
  });

  workerInstance.on('failed', (job, err) => {
    console.error(`[report-worker] job ${job?.id} falhou:`, err.message);
  });

  return workerInstance;
}
```

| report-worker → Prisma | Worker usa prisma direto (não tenantClient) pois roda fora de contexto de request; deve sempre filtrar por empresaId explicitamente |
