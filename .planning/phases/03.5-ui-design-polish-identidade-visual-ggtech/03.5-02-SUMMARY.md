# Phase 3.5 Plan 2: Sidebar Responsiva + Layout Wrapper Summary

**One-liner:** Sidebar extraida para componente isolado com drawer mobile (translate-x + overlay) e layout wrapper em page.tsx garantindo ausencia de overflow horizontal de 375px a 1280px+.

## O Que Foi Feito

Extracao do bloco `<aside>` inline do DashboardClient para um componente `Sidebar` dedicado com suporte a drawer mobile. Refatoracao do DashboardClient para consumir o novo componente e gerenciar o estado `sidebarOpen`. Ajuste do layout em page.tsx para fornecer o container flex correto e prevenir overflow-x em todos os breakpoints.

### Arquivos Criados

**apps/web/src/components/sidebar.tsx**
- Client component com props: `cameras`, `userName`, `userRole`, `isOpen`, `onClose`
- Desktop (lg+): `fixed inset-y-0 left-0` com `lg:relative lg:flex-shrink-0 lg:translate-x-0` — sempre visivel como elemento estatico no layout flex
- Mobile (<lg): drawer com `transition-transform duration-300 ease-in-out`; `isOpen ? translate-x-0 : -translate-x-full`
- Overlay `fixed inset-0 z-30 bg-black/50 lg:hidden` clicavel para fechar
- Botao X (lucide-react) com `lg:hidden` — visivel apenas no mobile

### Arquivos Modificados

**apps/web/src/components/dashboard-client.tsx**
- Importa `Sidebar` de `./sidebar` e `Menu` de `lucide-react`
- Adiciona estado `sidebarOpen: boolean` (padrao `false`)
- Remove bloco `<aside>` inline (145 linhas); substitui por `<Sidebar>` com 6 linhas
- Retorna Fragment (`<>`) sem wrapper div: `Sidebar` + `<main>` como filhos diretos
- Botao hamburger `data-testid="hamburger-btn"` com `lg:hidden` abre sidebar no mobile
- Toda logica de negocio (Socket.IO, classificacao, scroll pinning) preservada sem alteracoes

**apps/web/src/app/page.tsx**
- Div raiz: `overflow-hidden` adicionado (previne scroll horizontal no mobile)
- Header: `flex-shrink-0` adicionado (nao comprime com conteudo grande)
- Novo wrapper `div.flex.flex-1.overflow-hidden` envolve `<DashboardClient>` (fornece contexto flex para Sidebar + main)

## Estrutura do Drawer Mobile (Classes Tailwind)

```
<aside
  className="
    flex flex-col overflow-y-auto bg-ggtech-darkblue p-4
    fixed inset-y-0 left-0 z-40 w-72
    transition-transform duration-300 ease-in-out
    lg:relative lg:flex-shrink-0 lg:translate-x-0
    [isOpen ? translate-x-0 : -translate-x-full lg:translate-x-0]
  "
>
```

## Hierarquia de Layout Final

```
div.flex.h-screen.flex-col.overflow-hidden.bg-gray-50   (page.tsx)
  header.flex.flex-shrink-0...                           (top bar)
  div.flex.flex-1.overflow-hidden                        (body wrapper, page.tsx)
    <>                                                   (Fragment, DashboardClient)
      Sidebar (fixed mobile / relative desktop)
      main.flex.flex-1.flex-col.overflow-hidden.p-4
        div (hamburger + titulos)
        div.overflow-y-auto (feed de eventos)
    </>
```

## Decisoes Tomadas

1. **Breakpoint lg (1024px) para colapso:** O plano mencionava `<=768px` no texto descritivo mas usava `lg:` nas classes de exemplo. Foi mantido `lg` (1024px) por ser o breakpoint semantico correto para distinguir mobile/tablet de desktop em Tailwind.

2. **Fragment sem wrapper div no DashboardClient:** O plano original tinha um `div.flex.flex-1.overflow-hidden` dentro do DashboardClient. Ao mover esse wrapper para page.tsx, eliminamos o aninhamento duplo — page.tsx agora e o unico responsavel pelo container flex do body.

3. **lucide-react instalado em apps/web:** O pacote nao estava presente. Instalado com `pnpm add lucide-react --filter=@cargo-sentinel/web`. Versao `^1.21.0`.
