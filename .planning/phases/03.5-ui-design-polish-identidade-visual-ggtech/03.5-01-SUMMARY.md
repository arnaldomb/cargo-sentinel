# Phase 3.5 Plan 1: Design Tokens + Typography Summary

**One-liner:** Roboto e Open Sans carregadas via next/font/google com variáveis CSS --font-roboto/--font-open-sans injetadas no html element, eliminando CDN externo em runtime.

## O Que Foi Feito

Configuração completa de tipografia ggtech via next/font/google no root layout do app web. Elimina requisições a fonts.googleapis.com em runtime (threat T-3.5-01 mitigado), evita FOUT via font-display swap, e conecta as fontes ao sistema Tailwind via CSS custom properties.

### Arquivos Modificados

**apps/web/src/app/layout.tsx**
- Importação de `Roboto` e `Open_Sans` de `next/font/google`
- Instanciação com subsets latin, display swap, e variáveis CSS nomeadas
- `<html>` element recebe `className={${roboto.variable} ${openSans.variable}}`
- `<body>` mantém `font-sans` para herdar Roboto via Tailwind

**apps/web/tailwind.config.ts**
- `fontFamily.sans` alterado de `['Roboto', 'sans-serif']` para `['var(--font-roboto)', 'system-ui', 'sans-serif']`
- `fontFamily.heading` alterado de `['Open Sans', 'sans-serif']` para `['var(--font-open-sans)', 'system-ui', 'sans-serif']`
- Cores ggtech preservadas intactas: ggtech-darkblue, ggtech-blue, ggtech-lightblue

**apps/web/src/app/globals.css**
- Removida regra `html { font-family: Roboto, system-ui, sans-serif }` (string literal)
- Mantida regra `h1-h6` com `var(--font-open-sans)` via CSS custom property
- Sem nenhuma string literal de fonte restante

## Variáveis CSS Criadas

| Variável | Fonte | Pesos | Uso Tailwind |
|----------|-------|-------|--------------|
| `--font-roboto` | Roboto | 400, 500, 700 | `font-sans` |
| `--font-open-sans` | Open Sans | 600, 700 | `font-heading` |

## Decisões Tomadas

1. **Pesos de Roboto (400/500/700):** 400 para body text, 500 para labels e badges de status, 700 para ênfase — corresponde ao uso nos componentes do dashboard.

2. **Pesos de Open Sans (600/700):** Headings no dashboard usam `font-semibold` (600) e `font-bold` (700). Sem 400/500 para economizar bundle de fonte.

3. **globals.css simplificado mas não vazio:** A regra `html { font-family }` foi removida porque `body.font-sans` no layout já aplica `var(--font-roboto)` via Tailwind para todo o documento. A regra `h1-h6` foi mantida com `var(--font-open-sans)` para garantir que headings HTML nativos (sem classe Tailwind) também recebam Open Sans.

4. **display: swap em ambas:** Fallback imediato para system-ui enquanto as fontes carregam. Aceito conforme T-3.5-02 no threat model — risco mínimo de FOUT residual.
